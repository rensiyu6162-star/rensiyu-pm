04

关键交互设计决策

侧导航默认开启

目标用户以桌面端为主，侧导航可持续提供页面锚点，同时不占用内容区域宽度。支持一键切换顶导 / 收起侧栏，满足不同浏览习惯。

实习地图 Hover 框稳定驻留

解决"鼠标移到 tooltip 上卡片立即消失"的可用性问题。通过 CSS pointer-events 控制，只要鼠标未完全离开 hover 框，浮层就持续展示，避免反复触发造成的挫败感。

子页面回锚定位

从 AI能力全景 / 生活页返回时，页面自动定位到进入时的来源卡片（通过 URL hash + 元素 id 实现），避免用户迷失在首页顶部重新查找。

个人照片轮播

4 张生活照每 5 秒自动切换，支持手动前后翻页与点状指示器，平衡真实感与信息获取效率，避免静态单图的单薄感。

05

视觉设计决策

为什么选深色赛博朋克风格，而非常规极简白底？

  • 差异化记忆点：同类候选人网站多为极简白底，暗色科技感在面试官浏览多份作品集后具有显著记忆优势
  • 与 AI 产品方向高度契合：视觉语言传达"懂技术审美、有设计专业度"
  • 设计背景的直接体现：工业设计 + 交互设计背景的候选人，视觉品质本身就是能力的一种展示

视觉语言系统

  • 主色调：深紫黑背景 + 青色（#00f0ff）+ 品红（#ff00aa）双主色
  • 材质感：毛玻璃卡片（backdrop-filter blur）+ 网格背景纹理
  • 动态背景：魔法阵风格 Canvas 动画，增强科技氛围

排版策略

  • 信息层级通过字号、字重、颜色三维度区分，不依赖复杂装饰
  • 关键数字和标签用 clamp() 响应式字号，保证不同屏幕比例一致
  • 重要文案控制在 54ch 内，保证最佳阅读舒适度
06

技术选型与落地路径

为什么选纯静态 HTML/CSS/JS，不用框架？

  • 部署门槛极低：本地 python3 -m http.server 即可预览，上线只需拖拽文件夹
  • 维护成本低：内容调整直接编辑 HTML，无构建流程
  • 加载速度快：面试官点开时零等待，首屏体验即决策体验
  • AI 协作友好：静态代码对 AI 工具透明，便于与 Comate/Comate 持续迭代

与 AI 的协作分工

  • 我主导：用户分析、信息架构、内容文案、设计语言决策、交互逻辑定义
  • AI 负责：CSS 实现、动效代码、布局微调、批量文案替换与 Bug 修复

迭代方式

  • 基于面试官真实反馈持续微调文案与信息层级
  • 求职方向变化时同步更新定位文案与能力描述
  • 网站本质是动态求职材料，非一次性交付物
07

设计收益与反思

预期收益

  • 相比纯简历，信息传达更立体，减少面试前沟通成本
  • AI能力有具体落地案例支撑，减少面试官的信任成本
  • 视觉差异化提升记忆留存率，增加主动回访可能性

局限与后续优化方向

  • 内容较多，首屏信息密度仍可继续压缩
  • 移动端体验有待进一步打磨
  • 可加入访问统计，了解哪些模块被访问频率最高，指导下一轮迭代
